Configure AWS S3

  • Click on the "Create bucket" button.
  • Enter a unique bucket name (e.g., "myresumeo-forcloudwork"). Note: Bucket names must be globally unique across all existing bucket names in Amazon S3.
  • Select the AWS region where you wish to create the bucket.
  • Begin uploading your previously developed HTML and CSS files into the S3 bucket.
  • Click on "Properties" and ensure that 'Static web hosting' is enabled. If not, click on "Edit" and enable it.
  • In the "Index document" field, specify 'index.html' as the default index document.

Configure AWS CloudFront

  • Navigate to CloudFront in the AWS console. CloudFront acts as a Content Delivery Network (CDN) to deliver website content with lower latency and higher transfer speeds without making the S3 bucket publicly accessible.
  • In the "Origin Domain" section, select the S3 bucket that we created earlier, which is "myresumeo-forcloudwork".
  • In the "Origin access" section, select "Origin access control settings". This will generate a policy that we can use later for the S3 bucket.
  • In the "Viewers" section, select "HTTPS only" to ensure the secure protocol over HTTPS. The rest of the settings can be left as default.
  • Click "Create" and wait for the distribution to deploy.
  • After the distribution is successfully deployed, we will see a "Copy policy" box. This policy will grant the distribution access to our S3 bucket to read the objects since our bucket is not set to public.
  • Return to our S3 bucket and click on "Permissions" -> "Bucket Policy" -> "Edit", and paste the policy copied from CloudFront.
  • It takes around 5-10 minutes to deploy the distribution.
  • I encountered an issue when I tried to access the CloudFront URL from the distribution, as it gave me an 'Access Denied' error.
  • I reviewed the policy and updated the policy for the S3 bucket once more.
  • I encountered another error when trying to access other HTML pages.
  • Go to your CloudFront distribution in the AWS Management Console, navigate to the "Invalidations" tab, and create a new invalidation.

Set Up a Hosted Zone in Route 53

  • Click on "Hosted zones" in the navigation pane.
  • Click on the "Create hosted zone" button.
  • Enter your domain name (e.g., example.com) in the "Domain name" field.
  • Choose the appropriate type, usually "Public hosted zone".
  • Click "Create hosted zone"
  • After the hosted zone is created, click on it to view the details.
  • In the "Hosted zone details" section, note the names of the four DNS servers listed under "Name Servers".

Update GoDaddy DNS Settings

  • Sign in to your GoDaddy account and open your "My Products" page.
  • Locate your domain and click on the DNS button to the right of it.
  • Scroll to the "Nameservers" section and click on "Change".
  • Select "Enter my own nameservers (advanced)".
  • Select "I'll use my own nameservers".
  • Enter the Route 53 name servers that you noted earlier. Add each of the four name servers provided by Route 53.
  • Save your changes.

Verify DNS Propagation

  • DNS changes can take up to 48 hours to propagate fully, although it often happens much sooner.
  • To verify that the domain is pointing to Route 53, you can use a DNS checker tool like "What's My DNS" to see if the name server changes have propagated.

Setting up Lambda function

  • AWS Lambda is a serverless computing service provided by Amazon Web Services (AWS) that allows you to run code without provisioning or managing servers.
  • Choose "Author from scratch".
  • Enter a function name.
  • Select a runtime
  • Choose or create an execution role.
  • Select "Enable Function URL" . This allows you to create a dedicated HTTP(S) endpoint for your Lambda function. This feature enables direct HTTP access to your function without needing an additional service like API Gateway
  • Select "Cross-Origin Resource Sharing (CORS)" .This is a security feature implemented by web browsers that allows or restricts resources on a web page to be requested from another domain outside the domain from which the resource originated
  • This Lambda function interacts with a DynamoDB table named 'Cloudresume'. It retrieves an item with a primary key 'id' of '1', increments the 'views' attribute by 1, updates the item in the table with the new 'views' value, and returns the updated view count.
